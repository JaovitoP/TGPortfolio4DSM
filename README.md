## Data-Sars | 2023-1 
 ![logoDataSars](https://github.com/user-attachments/assets/f9542a8f-ed17-44d5-b500-685c1dc0cb19)
 
[Acesso ao GIT do Data-Sars](https://github.com/Grupo-Syntax-Squad/API)

### Empresa Parceira - Fatec (Cliente: Prof Fernando Massanori Ashikaga)

### Problema

A COVID Longa é uma condição que se refere aos sintomas que um paciente apresenta em após ter a COVID 19 como palpitações cardíacas, febre e dificuldade em respirar. Os dados relacionado à COVID Longa não foram tão divulgados quanto os da COVID 19 em si, e os portais de transparência, com dados da saúde, não apresentam de forma simples para a população, de modo que dificulta até para pessoas atuantes na área da saúde de encontrar estes dados.

### Solução

Para jornalistas da Rede Vanguarda que desejam acessar, visualizar e analisar dados da COVID Longa, o "Data-SARS" é um site que permite um fácil acesso a informações relacionadas a COVID Longa. Ao contrário de alguns sites que propagam Fake News e que não possuem filtros de pesquisa, o nosso produto fornece os dados de maneira que seja fácil de entender e de analisar os dados, uma vez que nossos dados são autênticos e possuímos uma área de interação minimalista.

### Tecnologias Utilizadas 

* Amazon AWS - Plataforma de serviços em nuvem utilizada para hospedagem, armazenamento e gerenciamento de infraestrutura do projeto.
* Flask - Framework web em Python utilizado para construir o back-end e gerenciar as rotas e APIs.
* HTML5 - Linguagem de marcação utilizada para estruturar as páginas web do front-end.
* CSS3 - Linguagem de estilo utilizada para estilizar as páginas web, garantindo uma interface visual agradável e responsiva.
* Python - Linguagem de programação utilizada para desenvolver a lógica de negócio do sistema e integrações.
* Visual Studio Code - Editor de código utilizado para escrever, editar e depurar o código do projeto.
* Docker - Plataforma de containers utilizada para criar ambientes de desenvolvimento e produção consistentes.
* Excel - Ferramenta utilizada para análise e manipulação de dados gerados pelo sistema.
* Figma - Ferramenta de design gráfico utilizada para criar protótipos e layouts das interfaces do sistema.

### Contribuições Pessoais 

Atuei na prototipação com Figma e criação do site com HTML e CSS, e na sua simplificação com o microframework Flask, do Python e também na adaptação de estilos para diferentes navegadores. Além disso, pesquisei junto à minha equpe por dados nos portais de transparência de alguns municípios no Vale do Paraíba- SP 

### Aprendizados Efetivos 

### Soft Skills

## CallGenie | 2023-2 

![callgenielogogit](https://github.com/user-attachments/assets/a01cec9a-f50f-40f5-9fbd-39039fa47e37)

### Empresa Parceira - Fatec (Cliente: Prof Cláudio Etelvino de Lima) 

[Acesso ao GIT do CallGenie](https://github.com/Grupo-Syntax-Squad/CallGenie)

### Problema
O prof Claudio, da Fatec, propôs a criação de um sistema para empresas e companhias que buscam implementar soluções para gerenciamento de Chamados de Serviços, de modo que facilite a comunicação entre cliente e suporte técnico.


### Solução

O "CallGenie" é um sistema CRUD (create, read, update, delete) que permite a melhor gestão dos chamados, contando com uma área de interação minimalista e documentos que auxiliam o usuário a usar o sistema. Neste projeto, implantamos o sistema para uma loja fictícia de informática, entretanto, nosso sistema pode ser implantado em qualquer área, contanto que exista a dificuldade de gestão dos chamados. 

### Tecnologias Utilizadas 

* Express - Framework web para Node.js utilizado para criar APIs e gerenciar rotas no back-end de maneira simples e eficiente.
* Javascript - Linguagem de programação utilizada para implementar a interatividade e funcionalidades dinâmicas no front-end.
* Typescript - Linguagem de programação baseada em JavaScript, com suporte a tipagem estática, utilizada para tornar o desenvolvimento mais robusto e organizado.
* React - Biblioteca JavaScript utilizada para criar interfaces de usuário dinâmicas e componentes reutilizáveis no front-end.
* MySQL - Sistema de gerenciamento de banco de dados relacional utilizado para armazenar e gerenciar os dados do projeto.
* Node - Ambiente de execução de JavaScript no lado do servidor, utilizado para desenvolver aplicações escaláveis e de alta performance.
* Sequelize - ORM (Object-Relational Mapping) para Node.js utilizado para interagir com bancos de dados relacionais, simplificando consultas e operações.
* HTML - Linguagem de marcação utilizada para estruturar as páginas web do front-end.
* CSS - Linguagem de estilo utilizada para estilizar e criar layouts responsivos e atraentes nas páginas web.
* Figma -  Ferramenta colaborativa de design utilizada para criar protótipos, wireframes e layouts de interfaces de usuário.

### Contribuições Pessoais 

Atuei na prototipação do site utilizando o Figma, no desenvolvimento do site inicialmente com Express de em seguida com o framework React, e também na estilização utilizando CSS

## Tupan | 2024-2 

 ![Banner](https://github.com/user-attachments/assets/edce9ea3-13ec-4363-b97f-64be715cfa92)

### Empresa Parceira - Tecsus 
Sobre a empresa: A TecSUS é uma empresa de tecnologia que realiza gestão de utilidades em água, energia e gases, a fim de reduzir custos, evitar desperdícios e antecipar reparos.

[Acesso ao GIT do Tupan](https://github.com/Grupo-Syntax-Squad/Tupan)

### Problema
A empresa Tecsus, especializada em IoT para utilidades, planeja desenvolver estações meteorológicas de baixo custo para monitoramento ambiental, utilizando sensores de vento, chuva, umidade, temperatura e pressão. Os dados serão enviados para um servidor, tratados e exibidos em um portal com relatórios, dashboards e conceitos matemáticos para engajar alunos do ensino médio. O objetivo é conscientizar sobre a importância do monitoramento ambiental e a prevenção de desastres naturais.

### Solução

O Tupan, projeto desenvolvido para solucionar o problema da Tecsus, tem como objetivo desenvolver um sistema de monitoramento de estações meteorológicas de baixo custo, capaz de receber e armazenar dados de sensores como direção e velocidade do vento, pluviometria, umidade, temperatura e pressão. O sistema gerará relatórios e dashboards com o histórico de leituras e enviará alertas em situações de risco. 

### Tecnologias Utilizadas 

* Typescript - Linguagem de programação baseada em JavaScript, mas com suporte a tipagem estática, utilizada para tornar o desenvolvimento mais robusto e evitar erros.
* Next - Framework baseado em React, utilizado para criar aplicações web modernas com renderização no servidor (SSR) e geração de sites estáticos.
* Postgresql - Banco de dados relacional open-source utilizado para armazenar e gerenciar os dados do projeto de forma eficiente. 
* Node - Ambiente de execução de JavaScript no lado do servidor, utilizado para criar aplicações escaláveis e de alto desempenho.
* Django - Framework web em Python utilizado para construir o back-end do projeto, com foco em segurança, escalabilidade e rapidez no desenvolvimento.
* Tailwind - Framework CSS utilitário utilizado para estilizar as interfaces do projeto de maneira rápida e consistente.
* Python - Linguagem de programação versátil utilizada para desenvolver o back-end e scripts auxiliares no projeto.
* Figma - Ferramenta de design colaborativa utilizada para criar protótipos e layouts das interfaces do sistema.
* Docker - Plataforma de containers utilizada para padronizar ambientes de desenvolvimento, facilitar a implantação e isolar dependências.
* Redis - Banco de dados em memória, utilizado para cache, filas e outras operações de alta performance no projeto.
* Azure - Plataforma de computação em nuvem da Microsoft utilizada para hospedar e gerenciar a infraestrutura do projeto.
* VisualStudio Code - Editor de código utilizado para desenvolver, depurar e colaborar no desenvolvimento do projeto.
* C - Linguagem de programação utilizada para desenvolver componentes de baixo nível ou aplicações com alta performance.
* Cypress - Framework de testes end-to-end utilizado para garantir a qualidade e o comportamento correto das aplicações web.
* Jest - Framework de testes em JavaScript utilizado para escrever e executar testes unitários e de integração.
* Swagger - Ferramenta utilizada para documentar e testar APIs de forma interativa e amigável.
* SonarQube - Plataforma de análise de código utilizada para garantir a qualidade e segurança do código, detectando bugs e vulnerabilidades.
